# GUARD-1275 Safety Portal Settings Prototype Design

## Overview
This prototype implements the UI and state management for Jira task GUARD-1275: "Safety: Request Portal Settings â€” Safety Toggle & Options". It provides a unified view containing both the Admin configuration interface and the resulting Public Request Portal interface to demonstrate how settings affect the end-user experience.

## Architecture & State Management
The prototype will be built as a single Next.js route (`/GUARD-1275-v1.0`) containing a unified view. 

State will be managed via a local React Context (`Guard1275Context`) to simulate the backend settings.

**State Interface:**
```typescript
interface PortalSettings {
  safetyEnabled: boolean; // The main toggle from GUARD-1275
  requiredFields: {
    attachments: boolean;
    location: boolean;
    asset: boolean;
    category: boolean;
    priority: boolean;
  };
}
```

## Components

### 1. `Guard1275Context`
- Holds the `PortalSettings` state.
- Provides functions to toggle `safetyEnabled` and toggle individual `requiredFields`.

### 2. `AdminView` (Left side or Top section)
- Simulates the CMMS Request Portal settings modal.
- **UI Elements:**
  - "Allow Safety Event creation" toggle switch.
  - "Request Options" section with checkboxes for Attachments, Location, Asset, Category, Priority.
- **Interactions:**
  - Toggling switches updates the `Guard1275Context`.

### 3. `PortalView` (Right side or Bottom section)
- Simulates the Public Request Portal (`ai-request-portal`).
- **UI Elements:**
  - Header: "Let's Report the Issue"
  - Tabs: "Work Order" | "Safety Event"
    - The "Safety Event" tab is **only visible** if `safetyEnabled` is true in the context.
  - Form: When "Safety Event" is selected, shows the Safety Event Details form.
    - Form fields (Attachments, Location, Asset, Category, Priority) show a red `*` (required indicator) based on the `requiredFields` state in the context.

### 4. `PrototypeLayout`
- A wrapper component that places `AdminView` and `PortalView` side-by-side (on large screens) or stacked (on smaller screens) so the user can see the effect of admin changes immediately on the portal.

## Data Flow
1. User (acting as Admin) toggles "Allow Safety Event creation" in `AdminView`.
2. `Guard1275Context` updates `safetyEnabled` to `false`.
3. `PortalView` re-renders and hides the "Safety Event" tab.
4. User toggles "Category" to required in `AdminView`.
5. `Guard1275Context` updates `requiredFields.category` to `true`.
6. `PortalView` re-renders and adds a red `*` next to the Category field in the Safety Event form.

## Styling
- Use Tailwind CSS.
- Match existing UpKeep styling (blue primary colors, gray backgrounds, standard card shadows) based on the provided screenshots.
- No real API calls; purely UI and local state.
